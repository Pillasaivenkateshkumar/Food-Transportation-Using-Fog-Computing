# 4-Minute Presentation and Demo Script

## 0:00 to 0:40 - Problem and Motivation

- Explain that vaccine and food cold chains fail when telemetry is delayed or ignored.
- Position fog computing as the way to make faster decisions near the route.

## 0:40 to 1:30 - Architecture

- Walk through sensors, fog analytics, backend queue, and live dashboard.
- Highlight that the fog node filters noise, detects anomalies, and reduces bandwidth before forwarding data.

## 1:30 to 3:10 - Live Demo

- Start with the dashboard overview and live status cards.
- Show a stable vaccine route versus a high-risk dairy route.
- Point out alerts, route progress, compliance score, and fog intelligence metrics.
- Mention the local buffering behaviour if backend connectivity is interrupted.

## 3:10 to 4:00 - Wrap-Up

- Summarise how the project addresses the module brief.
- Mention the queue-based backend, cloud-ready deployment assets, and GitHub repository.
- Close with the hardest part: balancing edge responsiveness with clean backend scalability.
